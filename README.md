# v6.0 チョッパー式というのに挑戦してみる。

v5.0 で考えたことは全部が無駄というわけでも無さそうだ。実は、ダイオードで、p-MOS が閉じて
いるときの電流を GND に長さずに回収して、入口に流しこむことで、効率が上がるらしい。チョッ
パー型の DCDC の原理だそうだ。回生エナジーを上手く使おうということだな。

ということで Falstad での練習が [こちら](./v6.0/falstad.txt)。
PWM は RL78/G24 を 16MHz 駆動にして 1000カウントで 16kHz で考えてる。2^n よりも多分、計算
しやすそうなので。

コイルはサガミ RTP8010-330M の予定。Falstad のコイルには抵抗は無いので自分で入れる。0.086
Ωとのこと。Elshop で TDK TSL1112RA-330K2R3 がセール対象で心魅かれるが、普段からほほコイル
を使ったことがなく33uH がどれくらい汎用性があるのかも分からないので、見送る。こちらの抵抗
は 標準値は不明だが最大で0.062Ω。サガミの 7313NC-330M は高性能だが、さすがにでかすぎる。

ショットキーは [11EQS03L](https://akizukidenshi.com/catalog/g/g108997/) で、逆飽和電流を
0.45V と 0.4V のときの T=25℃での電流値に、おおよそ合せ込んである。

pMOS は [2SJ334](https://akizukidenshi.com/goodsaffix/2sj334.pdf) で
βは単純な二次式による近似なので、データシートの全部に一致することはない。データシートは派
手な数値ばかり記載されているが、自分が欲しい 1A あたりのデータは、データシート 3ページ目の
左上 ID-VDS プロットである。ここから 

| VGS[V] | VDS[V] | ID[I] |
| --- | --- | --- |
| 2.5 | 0.135 | 1.0 |
| 3.0 | 0.06 | 1.0|

が得られる。Falstad の式では Vth 以下では電流が 0 になるので、ここはエイヤーで Vth=-1.4V
とした。それに対して β=8.35 となった。
[Falstad回路](./v6.0/2SJ334_falstad.txt)

一番右端のコンデンサは、パスコン。
ADC の C が 5pF と仮定する。インピーダンスは C に反比例。パスコンを入れて並列にしたら、そ
の比率でインピーダンスが低い方に流れる。

一方、ADC は 4096分解能。つまり,1波で 4096カウント分のノイズが入ったとしても、ノイズを
1/4096 まで減らせば 1カウントの誤差にまで減らせる。1/40960 まで減らせば 0.1 カウントだ。

そして 5pF の 40960倍 = 0.2uF だ。つまり 0.22uF で良いだろう。Falstad でも 0.1Ωのところの
電圧は安定しているように見える。そもそも ADC にパスコンなんて入れたこともないし、悪くなる
ことは無いだろう。

Fasltad で 1A を狙うようにデューティ比を調整しながら、電流や消費電力を測ってみた。
[計算シート](./v6.0/calc.gnumeric)

そしてDuty% (D), 電池電圧 (Vb), 電流(I) について、以下の近似式を得た。

```math
D = 87.382624  -26.930792 Vb -5.5315597 I 
``` 

今回 I=1A を狙っているので、

```math
D = 81.851064  -26.930792 Vb
```

が得られる。

どうせこの通りにはならないので、電流 (0.1Ωの電圧) が足りなければ 1カウント+, 越えてれば 1
カウント- みたいに補正値を与えれば良いだろう。

# v5.0 （没)

[こちら](https://github.com/Nobutarou/ArduinoDeAsobu/tree/main/NiMH_Charger)
で制作してきたが、リポジトリ自体がかなり大きくなってきたので、こっちに移った。そっちで検討
していた V4.0 は、なんだか苦労する割に報われない気がしたのでやめた。

v3.0 の改良を測ってみる。v3.0 の弱点は

- 充電中の電池電圧を測っていない
  - PWM でパチパチやってるときに、電圧測っちゃだめな気がしたため
- NchMOSFET を使ったため、電池の底の電圧を測定したが、多分これが精度低すぎ。
- 1A を狙ったが、実際には 70% くらいだった。

[こちら](https://github.com/Nobutarou/ArduinoDeAsobu/tree/main/LiIonCharger) 
ではリチウムイオン電池を充電してみたが、この時は PchMOSFET を使い、電池の頭の電圧を測定し
たが、値は安定しているように思う。

なので、今回は、PchMOSFET を使って LC フィルタで「まあこれなら電圧測っても大丈夫くらいだろ
う」と思える程度まで平滑化してみたい。平滑化するには PWM を極力高周波数で回すことが大事な
ので、RL78/G24 の最大32MHz 駆動で、99 または 255 カウントあたりの PWM, 周波数にして
125k-320kHz を使いたい。

li-ion のときの 
[2SJ334](https://akizukidenshi.com/goodsaffix/2sj334.pdf)
が 100kHz 台のパチパチに耐えられるかがポイント。

Vgs=-10V で Qg=110nC とあるので、Ciss=11000pF を採用したときのシミュレーションをする。
Falstad の P-mos の βは

```math
\beta = \frac{Y_{fs} }{Vgs-Vth}
```

Yfs が無いときは
```math
\beta = \frac{1}{R_{DS(on)} (V_{GS} - V_{th})}
```

でやってみると、そもそも理想状況でも全然電圧がでない。コイルは交流成分からしたら抵抗だから、
電圧がぐっと下がってどうにもならない。
https://x.gd/wddPF

そもそも構造的欠陥。

でコイルを止めてショットキーを入れるとなんか整流できてる。
https://x.gd/BBFU5

C で決まるローパスの周波数はこうらしい。手持ちだと 220uF が丁度良い感じ。PWM の切替えも数
百 usec で落ち着くかんじ。
```math
fc = \frac{1}{2 \pi R C}
```

で Ciss=11000pF 入れると全然だめ。オンにできないようだ。125kHz に落としてもどうにもならな
い。逆に 220uF は 125kHz でも良い感じでフィルタできる。

ところでダイオードを入れたが 0.47Ωの R を 2個の 0.22Ωに分けても良いみたいだ。ただどのみ
ち Ciss=11000pF ではどうにもならない。あと PWM を 64kHz とかにすると 220uF でも足りなくな
りがち。
https://x.gd/201CP

さて Ciss だけど実は Ciss(Vgs=0V)=3300pF, Ciss(Vgs=10V)=11000pF である。なので、3300pF で
成立しなかったら、もうどうにもならない。Ciss 無しで 1.65V くらいのようだが、3300pF を入れ
ると、1.42V くらいまで下がってしまう。

ただ 3.3V ではどうだろうか。Ciss(Vgs) は凸な関数と Gemini が言ってるので、なんとなく、
```math
a+b Vgs ^n
``` 
みたいな Johnson-cook な感じが合うような気がする。
次元がおかしいので Vgs は電圧ではなく、基準電圧との比率とすべきだが、細かいことはいいだろ
う。
a=Ciss(Vgs=0) で良いが b と n は分かりよ
うがないので、とりあえず n=0.5 とする。b=2435[pF] となる。
この仮定だと Ciss(Vgs=3.3V)=7723pF である。はい、全然だめ。


では 
[2SJ496](https://akizukidenshi.com/goodsaffix/2sj496.pdf) 
はどうだろうか。Yfs=5, Vth=2, Vgs(ID=2.5A)=2.7Vくらい（ID-Vgs プロットから) なので β=7.14
Ciss に最小値の 600pF の 2.34 倍, 1404pF を入力してみる。Qgs が載ってないから超適当.
で 1.4V くらいは出る。

だめだ。この方法は無理だ。最初の PWM を直接当ててる方がまだまし。

というか、何をどうしようが、3.3V 電源から 1A 流す時点で全体の消費電力は 3.3W。電池が 1V の
時は 2.3W を捨てて、電池が 1.5V のときは 1.8W を捨ててると言うこと。

変わらない。
